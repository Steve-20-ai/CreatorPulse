# 可解释匹配算法

## 设计目标

匹配引擎不是寻找“粉丝最多的人”，而是在预算与风险约束下，找到与产品场景、目标受众和营销目标更匹配的达人，并输出可审核的理由。

## 两阶段决策

### 第一阶段：硬筛选

候选达人满足以下条件后进入排序：

1. 品牌安全评分不低于阈值（默认 `0.78`）。
2. 报价不超过单达人预算。
3. 与活动目标场景存在最低相关度。
4. 中国市场默认要求达人第一受众市场为 `CN`，避免把只有海外受众证据的达人混入中国候选池。

每个被排除的达人都会保留明确原因，避免黑盒丢弃。

代码保留 `requireMarketMatch: false` 选项，供未来具备目标市场受众占比证据、并经过人工审核的跨境实验显式启用；当前 Web 演示不默认开放该模式。

### 第二阶段：多目标评分

```text
综合分 =
28% 场景匹配
+ 22% 受众市场匹配
+ 14% 内容质量
+ 10% 互动质量
+ 12% 转化潜力
+  6% 成本效率
+  8% 履约与品牌安全
- 风险惩罚
```

权重可根据活动目标调整，但 MVP 默认权重保持固定，确保演示结果可复现。

## 三个差异化输出

### 1. 场景图谱

达人主场景、次要场景与产品场景标签共同计算匹配度。生产环境可将标签替换为“产品能力—真实任务—内容表达”的图结构和向量检索。

### 2. 证据链

系统按实际贡献排序，返回最重要的三条理由，并附上原始证据摘要。展示层不会生成数据集中不存在的统计事实。

### 3. 置信度与数据缺口

置信度由关键字段完整度与数据来源共同决定：

- 字段越完整，置信度越高。
- 合成数据自动降低置信度上限。
- 缺失点击、转化或报价时，明确列出需要补充的数据。

## 生产升级路径

1. 使用历史合作标签训练 Learning-to-Rank 模型。
2. 使用约束优化同时处理预算、市场覆盖、达人去重和竞品冲突。
3. 通过分层实验估计 AI 推荐相对人工选人的增量。
4. 定期监控模型漂移、地区偏差和品牌安全误报。
